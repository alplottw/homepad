# homepad

Кастомная **домашняя страница** Chrome (расширение, заменяющее new tab page).

Цель проекта — превратить страницу новой вкладки Chrome в полезный «дом» в браузере: начать со знакомого Google-подобного вида и постепенно добавлять панели и виджеты под реальные сценарии. Сделано на чистом Manifest V3, без фреймворков и сборки — легко форкать и допиливать под себя.

> English version: [README.md](./README.md)

## Что уже умеет

- **Главная часть** — логотип Google, строка поиска и сетка часто посещаемых сайтов (как у дефолтного NTP Chrome).
- **Сайдбар слева, верхняя секция** — список открытых вкладок (favicon + заголовок). Кнопка `+` сохраняет вкладку в раздел «Посмотреть позже». Список живой: обновляется при открытии/закрытии вкладок без перезагрузки страницы.
- **Сайдбар слева, нижняя секция** — сохранённые ссылки. Клик по заголовку открывает страницу в новой вкладке, кнопка `×` удаляет элемент. Сохраняются локально (`chrome.storage.local`).

## Установка (Developer mode)

1. Открыть `chrome://extensions/`.
2. Включить **Developer mode** (тумблер в правом верхнем углу).
3. Нажать **Load unpacked** и выбрать папку этого проекта.
4. Открыть новую вкладку (`Cmd+T` / `Ctrl+T`).

## Структура

```
manifest.json    — Manifest V3, override newtab
newtab.html      — разметка страницы
newtab.css       — стили
newtab.js        — логика (открытые вкладки, saved, top sites)
icons/           — иконки расширения (заглушки)
```

## Технические детали

- Manifest V3, без background service worker — вся логика работает в самой странице newtab.
- Разрешения: `tabs`, `topSites`, `storage`, `favicon`.
- Хранилище: `chrome.storage.local`. Синхронизации между устройствами нет (по требованию).
- Live-обновления: подписки на `chrome.tabs.onCreated/onRemoved/onUpdated/onMoved/onAttached/onDetached/onReplaced` (дебаунс 100 мс) и `chrome.storage.onChanged` для синхронизации между несколькими открытыми new tab.
- Иконки `icons/icon-*.png` — временные плейсхолдеры, можно заменить на финальные.
